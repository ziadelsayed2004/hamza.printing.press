## 1. متطلبات التشغيل الأساسية
قبل البدء، تأكد من امتلاك:
* خادم **Hostinger VPS** يعمل بنظام تشغيل **Ubuntu** (يُفضل Ubuntu 20.04 LTS أو أحدث).
* الاتصال بالخادم عبر الـ SSH بصلاحيات الـ Root (أو مستخدم يملك صلاحيات `sudo`).
* اسم نطاق (Domain Name) موجه إلى عنوان IP الخادم الخاص بك (اختياري، يمكنك استخدام الـ IP مباشرةً إذا لم يتوفر نطاق).

---

## 2. خطوة واحدة للنشر والتشغيل (Single Command Deploy)

قم بالاتصال بخادمك عبر الـ Terminal، ثم قم بتشغيل السطر البرامجي التالي مباشرة لتحميل الكود وتنصيب النظام بالكامل:

```bash
git clone https://github.com/ziadelsayed2004/hamza.printing.press.git hamza-press && cd hamza-press && chmod +x deploy.sh && sudo ./deploy.sh
```

*(تنويه: إذا كان الرابط الخاص بالمستودع مختلفاً أو خاصاً، قم بعمل clone للمستودع بالطريقة المعتادة ثم ادخل للمجلد وشغّل السكربت عبر `./deploy.sh`)*

---

## 3. ماذا يفعل سكربت التثبيت التلقائي؟
عند تشغيل [deploy.sh](file:///d:/Projects/BookStore%20Manager/Book-Store-Public/deploy.sh)، سيقوم بالعمليات التالية تلقائياً دون تدخل منك:

1. **تحديث الحزم الأساسية:** تنصيب الأدوات اللازمة مثل `Node.js v20`, `Nginx`, `SQLite3`, `Git`, `PM2`, و `UFW Firewall`.
2. **طلب نطاق العمل (Domain Configuration):** سيسألك عن اسم النطاق الخاص بك، وفي حال عدم إدخاله سيقوم بالاستعلام عن IP الخادم الخارجي وتكوين الإعدادات بناءً عليه.
3. **توليد المتغيرات البيئية الآمنة (.env):** نسخ الملف النموذجي وإنشاء مفتاح تشفير عشوائي وقوي جداً لجلسات العمل (`SESSION_SECRET`) بدلاً من المفاتيح الافتراضية، وتهيئة التطبيق على وضع الإنتاج (`production`).
4. **تنزيل حزم البرمجة:** تثبيت حزم الـ `npm` للواجهة الخلفية والأمامية معاً.
5. **بناء الواجهات (Vite Build):** تصدير ملفات React لتعمل بكفاءة وسرعة عالية.
6. **تهيئة قاعدة البيانات:** إنشاء قاعدة بيانات SQLite الجديدة وتفعيل جداول النظام وضخ بيانات الأدوار والمدير الافتراضي (Super Admin) بصلاحياته الكاملة.
7. **إعداد الصلاحيات والأمان:** قفل ملفات النظام وفتح صلاحيات الكتابة فقط لمجلد التخزين `storage/` الذي يحتوي على قاعدة البيانات والملفات المرفوعة.
8. **تشغيل وإدارة التطبيق في الخلفية (PM2):** تشغيل الخادم كخدمة مستمرة تعمل وتُقلع تلقائياً عند إعادة تشغيل السيرفر.
9. **إعداد خادم الويب (Nginx Reverse Proxy):** ربط منفذ التطبيق الداخلي (5000) بالمنفذ الخارجي العام (80) لتصفح الموقع مباشرة.
10. **تأمين الاتصال (Firewall):** إعداد جدار الحماية UFW للسماح بمرور حركة الويب وطلبات الـ SSH فقط.
11. **جدولة النسخ الاحتياطي التلقائي (Cron Job):** جدولة عملية حفظ النسخ الاحتياطية يومياً الساعة 2:00 صباحاً.

## 4. نظام النسخ الاحتياطي التلقائي (Automatic Database Backups)

قاعدة البيانات في النظام من نوع **SQLite** (عبارة عن ملف محلي مستقل داخل مجلد `storage/database.sqlite`). لحماية بياناتك من الضياع، قمنا بتفعيل **طبقتين من النسخ الاحتياطي التلقائي**:

### الطبقة الأولى: النسخ الاحتياطي على مستوى الخادم (Cron Job - مستحسن وآمن جداً)
تم إعداد مهمة مجدولة على الخادم (Cron Job) لتشغيل سكربت آمن [backup-db-safe.sh](file:///d:/Projects/BookStore%20Manager/Book-Store-Public/scripts/backup-db-safe.sh) **يومياً الساعة 2:00 صباحاً**.
* **ميزة الأمان:** يستخدم السكربت أمر `.backup` المدمج في SQLite3، مما يضمن نسخ قاعدة البيانات بشكل متوافق برمجياً وتجنب أي تلف حتى لو كان هناك مستخدمون يكتبون في قاعدة البيانات في نفس اللحظة.
* **إدارة المساحة:** يقوم السكربت تلقائياً بالبحث عن النسخ الاحتياطية التي مضى عليها أكثر من **30 يوماً وحذفها** للحفاظ على مساحة قرص الـ VPS.
* **المسار:** تُحفظ النسخ الاحتياطية في المجلد المخصص: `/var/www/hamza-press/storage/backups/`.

### الطبقة الثانية: الجدولة الداخلية للتطبيق (Node.js Scheduler)
يحتوي التطبيق داخلياً في ملف [backupScheduler.js](file:///d:/Projects/BookStore%20Manager/Book-Store-Public/server/modules/admin/backupScheduler.js) على كود يقوم أيضاً بمحاولة أخذ نسخة احتياطية إضافية يومياً الساعة 2:00 صباحاً كخط دفاع ثانٍ في حال تعطل خدمة الجدولة الخاصة بالنظام.

### التشغيل اليدوي للنسخ الاحتياطي:
إذا أردت أخذ نسخة احتياطية يدوية فوراً في أي وقت، يمكنك تشغيل هذا الأمر داخل مجلد المشروع:
```bash
npm run backup
```
أو تشغيل سكربت الـ Bash مباشرة:
```bash
bash scripts/backup-db-safe.sh
```

## 5. كيفية استعادة قاعدة البيانات من نسخة احتياطية (Restore Database)

في حال حدوث خطأ أو رغبتك في العودة بنظام التشغيل إلى تاريخ سابق، يمكنك استعادة أي نسخة احتياطية بسهولة عبر تطبيق الخطوات التالية:

1. **إيقاف تشغيل النظام مؤقتاً:**
   ```bash
   pm2 stop bookstore-manager
   ```

2. **عرض النسخ الاحتياطية المتوفرة واختيار إحداها:**
   ```bash
   ls -la storage/backups/
   ```

3. **نسخ ملف النسخة الاحتياطية المختار ليحل محل قاعدة البيانات النشطة:**
   *(استبدل اسم الملف بالاسم الفعلي للنسخة التي تريد استعادتها)*
   ```bash
   cp storage/backups/backup_2026-07-09_02-00-00.sqlite storage/database.sqlite
   ```

4. **إعادة ضبط صلاحيات الملف (هام لضمان قدرة النظام على القراءة والكتابة):**
   ```bash
   chmod 777 storage/database.sqlite
   ```

5. **إعادة تشغيل النظام:**
   ```bash
   pm2 start bookstore-manager
   ```

## 6. طريقة تحديث الكود مستقبلاً (Seamless Updates)

نظراً لأن قاعدة البيانات والملفات المرفوعة والنسخ الاحتياطية تعيش جميعها داخل المجلد المعزول `storage/` وهو مجلد تم إدراجه في تجاهل الجيت (`.gitignore`)، يمكنك تحديث الكود البرمجي للموقع وسحب الميزات الجديدة من GitHub في أي وقت دون الخوف من فقدان أو الكتابة فوق بياناتك.

لتحديث النظام، قم بتشغيل الأوامر التالية بالتوالي:

```bash
# 1. الدخول لمجلد المشروع
cd /var/www/hamza-press/

# 2. سحب التعديلات الجديدة من GitHub
git pull origin main

# 3. تحديث الحزم البرمجية الجديدة
npm install
npm install --prefix client

# 4. إعادة بناء ملفات واجهة المستخدم (Frontend)
npm run build

# 5. تطبيق أي تعديلات جديدة على هيكل الجداول بقاعدة البيانات (Migrations)
npm run db:migrate

# 6. إعادة تشغيل التطبيق في PM2 لتطبيق التغييرات
pm2 restart bookstore-manager
```
