# 077 — Legacy Feature Parity Deep Audit

## Goal
Ensure the new platform covers the old system and adds the requested upgrades.

## Scope
- Compare legacy/current features:
  - books
  - branches/outlets
  - invoices
  - balances
  - notifications
  - inventory
  - exports
  - roles
- Create a parity matrix:
  - old behavior
  - new behavior
  - status
  - missing gap
  - fix owner step

## Acceptance
- No important old feature lost.
- Gaps become explicit tasks or fixed in this step if small.
