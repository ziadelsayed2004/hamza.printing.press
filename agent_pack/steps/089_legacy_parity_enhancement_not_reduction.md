# 089 — Legacy Parity Enhancement Not Reduction

## Goal
Verify that modernization improves the old platform and does not reduce core value.

## Scope
- Read old repo notes/reports and current code.
- Confirm old value exists or improved:
  - balance.
  - notifications.
  - inventory warnings.
  - invoice operations.
  - shipping status.
  - exports.
  - dashboard summary.
- Explicitly confirm removed features are only:
  - import pipeline.
  - installments/payment plans.
  - old weak UI identity.
- Open small fixes inside this step only if they are directly required for parity and safe.

## Acceptance
- Report lists kept, improved, removed-by-decision, and still-missing items.
- No old important business behavior is silently lost.
